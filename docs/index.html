The Story

Why pseudocode
deserves to run.

There's a certain irony in the way educators handle pseudocode — they teach it as the universal language of logic, then immediately disqualify it from exams because it "isn't real code." That contradiction stuck with me. If pseudocode is expressive enough to explain algorithms to humans, why shouldn't it be expressive enough to explain them to machines? Pulsar is the direct answer to that question: a statically typed language where let score as number be 95 is valid, executable code. Not a sketch. Not a stepping stone. The real thing. The goal was always to let beginners — especially those making the jump from Scratch's building blocks to the syntax-heavy world of Swift or Python — write programs in language they already understand, without spending their first weeks memorizing colons, arrows, and bracket conventions. You immerse in the logic. The language handles the rest.

The name and the logo came together almost accidentally. I started with the aesthetic: a dark gradient background, stars, something that felt like space. Then I needed something in that universe that was predictable — something you could calculate, something intuitive. Pulsars fit perfectly. A pulsar is a rotating neutron star that emits regular, measurable bursts of radio waves; you can calculate exactly when the next pulse will arrive. That's the same guarantee Pulsar the language makes. You shouldn't need to read a specification document to understand how it works — after a few programs, the pattern becomes obvious, and you know what comes next. The constellation effect in the logo connects that same idea: individual stars forming a shape that, once you see it, you can't unsee. That's what I want Pulsar to feel like — not a language you study, but one you just start to recognize.

Features

What makes Pulsar different.

🔤
English-first syntax

Keywords read as sentences. as long as, go through each, is greater than — no symbols to memorize.

🔒
Static typing

Every variable, parameter, and return value has a declared type. Errors are caught before the program ever runs.

⚡
Transpiles to JavaScript

Pulsar compiles down to clean, readable JavaScript — making it easy to run anywhere Node.js runs.

📦
Groups & collections

Declare structured data with group, typed lists with list containing, and maps with map linking.

🛡️
Smart optimizer

Constant folding, dead code elimination, and algebraic simplifications run automatically before code generation.

🧪
100% test coverage

Every pipeline phase — parser, analyzer, optimizer, generator — is fully tested with a comprehensive suite.

Static Analysis

What Pulsar catches at compile time.

✦
Type mismatch on assignment Assigning a string to a number variable is a compile error
✦
Argument type checking Every call site is verified against declared parameter types
✦
Argument count matching Too many or too few arguments are caught before runtime
✦
Return type enforcement A function declared to output a number cannot return a string
✦
Scope resolution Undeclared and redeclared identifiers are flagged with line numbers
✦
Boolean-only conditionals if and as-long-as require boolean expressions — not numbers
✦
stop / output placement stop must be in a loop; output must be in a function
✦
Duplicate group fields A group cannot declare the same field name twice
✦
List homogeneity All elements in a list literal must have the same type
✦
No calling non-functions Attempting to call a variable as a function is a compile error

Swift 
// list_stats.swift
func sum(scores: [Double]) -> Double {
  var total: Double = 0
  for score in scores {
    total += score
  }
  return total
}

func max(scores: [Double]) -> Double {
  var highest: Double = 0
  for score in scores {
    if score > highest {
      highest = score
    }
  }
  return highest
}

let scores: [Double] = [72, 95, 88, 61, 100]
print(sum(scores: scores))
print(max(scores: scores))

Swift 
// student_report.swift
struct Student {
  var name: String
  var gpa: Double
  var year: Int
}

func standing(year: Int) -> String {
  if year == 1 { return "Freshman" }
  else if year == 2 { return "Sophomore" }
  else if year == 3 { return "Junior" }
  else { return "Senior" }
}

func isHonors(gpa: Double) -> Bool {
  return gpa >= 3.5
}

func report(name: String, gpa: Double, year: Int) {
  print(name)
  print(standing(year: year))
  print(isHonors(gpa: gpa))
}

// operators_and_loops.swift
func countdown(start: Int) {
  var current: Int = start
  var done: Bool = false
  while !done {
    print(current)
    current -= 1
    if current <= 0 {
      done = true
      break
    }
  }
}

func inRange(x: Int, low: Int, high: Int) -> Bool {
  return x >= low && x <= high
}

let x: Int = 10
let y: Int = 3
let diff: Int = x - y
let product: Int = x * y
let check: Bool = x != y
print(inRange(x: x, low: 1, high: 10))
